# PromptHub Top20 真实效果验证

验证日期：2026-06-21

## 验证说明

- 本报告使用当前 Codex/GPT 会话，对每条 Prompt 进行一次固定输入、单轮生成测试。
- “实际输出”是本次模型直接生成的结果，不是预先写在 Prompt 数据里的示例。
- 本次没有调用 Claude、Gemini 或 DeepSeek，因此只能证明单模型首轮可用性，不能替代多模型复测与真实用户反馈。
- 对篇幅较长的结果，报告保留能够判断质量的核心输出段落，而不是为了展示长度粘贴全部文字。
- 分类依据是实际输出是否可直接使用、是否节省工作时间、是否需要大量返工，以及是否与其他 Prompt 重复。

## 8. 程序报错根因分析

**测试输入**

环境：Next.js 15 App Router。预期：首页正常水合。实际：控制台提示Hydration failed。组件在服务端和客户端都渲染`new Date().toLocaleTimeString()`。刷新必现。

**实际输出**

假设按概率排序：

1. 服务端与客户端生成时间不同，HTML文本不一致。验证：暂时替换为固定字符串，错误应消失。
2. 服务端与浏览器时区不同。验证：记录两端时区与格式化结果。
3. 浏览器扩展修改DOM。无痕窗口验证，概率较低。

最可能根因是渲染阶段调用非确定性时间函数。最小修复：服务端传入固定ISO时间；若必须显示客户端当前时间，则先渲染占位符，在`useEffect`中设置，并避免影响关键布局。回归测试应覆盖首次加载无水合警告、固定时区快照和禁用JavaScript时的占位内容。

**结果分析**：根据证据正确排序假设，给出最小验证、根因、两种修复路径和回归测试，没有盲猜。

**缺陷**：没有直接给出示例代码。

**优化方案**：增加“输出最小补丁，但必须先说明补丁适用条件”。

**结论：优秀**

## 19. 考试错题复盘

**测试输入**

**实际输出**

错误类型判定为“方法选择与漏算”：3/5只计算了第一次抽到红球，没有覆盖第一次白、第二次红的情况。正确思路用对立事件：至少1红 = 1 - 两次全白 = 1 - (2/5 × 1/4) = 9/10。记忆提示：“至少一个，优先想一个都没有。”随后生成3道从“不放回同类题”到“至少两种颜色”的递进练习，不显示答案。

**结果分析**：错误诊断准确，解释简洁，练习遵守不提前给答案的要求。

**缺陷**：没有先让学习者解释自己的思路，互动诊断深度有限。

**优化方案**：先追问一步“你为什么选择3/5”，再决定错误类型和练习难度。

**结论：合格**
