# Workflow Content Standard

Every canonical Workflow page must help a user answer three questions:

1. What problem does this solve?
2. How do I execute it?
3. What will I have when I finish?

## Required Order

1. Workflow name
2. One-sentence value
3. Verification status and evidence boundary
4. Problem
5. Best-fit audience
6. Required inputs
7. Model assessments
8. Workflow steps
9. Core Prompt
10. Example input
11. Example output
12. Expected outputs
13. Common mistakes
14. Limitations
15. FAQ
16. Related case or Workflow

## Evidence Rules

- Model stars, rankings, and best-model claims require a documented comparison test.
- Time-saved claims require a recorded baseline, sample size, and test method.
- User outcomes and metrics require a traceable source or original research record.
- Example outputs must be labeled as examples unless reproduced in a verification run.
- Missing evidence must use `unverified`, `source-linked`, `compatible`, or `not-tested`.
- A blank evidence field is better than an invented value.

## Optional Fields

The following fields appear only when evidence exists:

- Time saved
- Success rate
- Model score
- Downloadable template
- User outcome
- Revenue or conversion impact

## Publishing Gate

A Workflow may be published as `unverified` when its method is useful and its evidence boundary is explicit. It may be labeled `verified` only after the verification protocol, saved outputs, failure analysis, and human review are complete.
